说明
①"信息头"是我自己取的称呼，主要是方便理解
②虽然很多地方可以不遵守 JS语法规则
③但还是请按格式书写或修改信息头
④JSON不能直接回车换行，所有信息都是一行
⑤字符串内有些字符是需要使用\号转义的
⑧源文件，指保存在【file:///sdcard/自动精灵】里的zjs文件，本质就是txt文本，可以直接编辑

信息开始{
①信息头的开始符号
②一个键值对象，一个属性就是一个精灵的参数
③参数出现顺序，由新增/修改参数时的时间为准
④参数的顺序不影响脚本执行 (键值属性是无序的)
⑤每个键值对之间用英文逗号分隔 (JS基础知识)

等待时间　"delay":"haha",
①全局设置中的默认等待时间
②默认值为 ""　代表空白未填写
③默认类型是文本，但也可以改为数值类型
④所以 "5" 和 5，效果是一样的
⑤如果修改的值不合法，会按照默认的1秒执行

等待单位　"delayUnit":100,
①默认等待时间的单位
②值为0，代表　毫秒
③值为1，代表　秒
④值为2，代表　分
⑤如果修改的值不合法，会恢复为默认单位 秒
⑥没什么值得玩的，所以我没怎么折腾

重复次数　"repeatCount":有多少爱可以重来,
①动作列表的循环次数
②默认值是 -1 (表示没有填写)
③修改为 0 则是无限循环
④正常编辑脚本，值必须是正整数 (会有验证)
⑤胡乱修改源文件，非法值也不会导致脚本错误
⑥错误的值会自动改为-1 (猜测的)，即默认1次

失败暂停　"pauseOnFail":false,
①动作运行失败后，是否暂停脚本的执行
②强迫你每个动作都需要有完善的应对措施
③默认是 true，关掉则是 false
④我一般都会关掉(不打勾)，即 false
⑤错误的参数，会恢复为默认 true

动作数量　"count":13800138000,
①报告脚本使用了多少个动作
②包含：动作监听、全局监听、子动作、子脚本
③默认值为数值类型 (整数型)
④但现在你可以自由修改
⑤例如修改成 "abcd" 也不会影响脚本运行
⑥除了装逼没多大作用

最小版本　"minVerCode":99999999,
①运行脚本最低需要什么精灵版本
②最小-版本-代号，数值类型
③运行脚本时会检查精灵版本计算是否比该值高
④可以改为任意值，值会变成NaN，等于数值0
⑤精灵2.12.0版本的代号是61 (每次升级迭代+1)
⑥2.13.0版本，代号是2013000 (该格式沿用至今)

最小版本　"minVerName":"222.33.44-GG",
①运行脚本最低需要什么精灵版本
②最小-版本-名称，字符类型
③主要就是展示作用 (显示在脚本详情里)
④格式一："数字.数字.数字"
⑤格式二："数字.数字.数字-英文"
⑥格式错误时，无法显示脚本详情 (不影响运行)

推荐宽度　"screenWidth":"八嘎",
①默认的屏幕宽度，电脑端称为推荐宽度
②默认为数值类型
③修改为0或者删除，会显示为 "通用分辨率"
④修改为任意值都行 (不是特殊字符就行)
⑤由于只是推荐，所以不会影响脚本运行

推荐高度　"screenHeight":0,
①默认的屏幕宽度，电脑端称为推荐高度
②默认为数值类型
③可为0，但是否变"通用分辨率"是由宽度决定的
④可删除，脚本详情中会直接不显示
⑤修改为任意值都行，不是特殊字符就行
⑥例如字符："文本"　例如数组：[1,2,"haha"]

推荐 dpi 　"screenDpi":0.123456789,
①默认的屏幕密度，电脑端貌似直接为1
②强制只能为数值类型 (虽然负值也合法)
③修改为0或者删除，都视为"兼容所有分辨率"
④参数错误会导致脚本详情异常，但可正常运行
⑤小小科普：精灵的点击坐标中查看x轴，100%切换为dp，假设变为360，屏幕宽度1080除以360=3即为dpi

保存宽度　"lastSaveScreenWidth":1080,
①最后一次编辑时保存的宽度
②如果横屏状态下保存脚本(选择当前分辨率)，宽度将大于高度
③横屏状态下保存脚本，"推荐宽度"依旧不会和"推荐高度"调换
④所以猜测"推荐分辨率"是开启悬浮窗时就读取设备信息的 (启动时精灵是竖屏)
⑤而"最后保存分辨率"，是根据实时状态读取的 (更能反映真实状态)

保存高度　"lastSaveScreenHeight":天高任我飞,
①最后一次编辑时保存的高度
②类似"推荐高度"，"保存分辨率"可以任意修改

保存 dpi 　"lastSaveScreenDpi":随便改,
①最后一次编辑时保存的密度
②默认为数值类型，但可以修改为任意值
③尚不清楚保存宽度、高度、密度有什么作用

保存版本　"savedVerCode":9926030,
①最后一次编辑时保存的精灵版本
②保存-版本-代号，数值类型
③可以改为任意值，值会变成NaN，等于数值0
④既然值为0，等于所有精灵版本都可以运行
⑤猜测：由于"最小版本"并不准确，所以需要这个"推荐"的版本提醒，下一个词条会详细讲解

保存版本　"savedVerName":"黄金版2.26.3",
①最后一次编辑时保存的精灵版本
②保存-版本-名称，字符类型
③可以修改为任意值，基本没有任何影响
④【起因】使用一个动作类型，精灵会记录至少需要xx版本才能支持
【问题】但该动作内部的某个参数或者某种写法格式，需要更高版本才支持，但精灵不会记录下来
【后果】使用低版本精灵运行这个脚本时，看不到该参数或功能。或者参数格式不被支持，可以运行但没预期的效果
【对策】新增本词条来记录"保存版本名称"，提醒用户尽量使用和脚本作者一样的精灵版本来规避环境差异导致的BUG

脚本描述　"description":"花里胡哨只会害了你",
①描述脚本的用途、写教程说明、推广宣传
②没啥好说，本来就是允许你任意写东西的地方
③特殊字符注意添加转义符就行

手势变形　"gestureMatrix":"瞎J8写",
①全局手势变形，对所有手势动作有效

③具体百度 css mateix 进行了解

是否加密　"encrypt":false,
①脚本是否加密
②true 为加密，false 为不加密
③加密需要通过市场，经过服务器加密再下载
④源脚本里添加本字段就可实现加密！！！
⑤加密后会阻止用户使用编辑模式运行
⑥改为false或者直接删除字段，则不加密
⑦记住：只是决定是否可编辑！不是破解

限制用户　"encryptUserId":"sub_12_1g23upker",
①加密后，指定可以运行脚本的用户ID
②字面翻译是：加密-使用者-id
③正常的加密方法必须通过市场 (市场+分享)
④下载脚本时服务器加密脚本同时添加本字段
⑤只会写入1个 id，且谁下载就写入谁的 id
⑥脚本运行时将检验当前的精灵用户id
⑦ sub 开头的id 是精灵后期推出的子帐号
⑧可以修改本字段，实现指定用户才能运行
⑨可以删除本字段，实现所有用户可用

过期时间　"encryptLimitUseTime":1659592615010,
①加密后，可指定过期之后无法运行脚本
②数值类型，时间戳，通常13位数，具体可百度
③当值不是一个合法数值时，会提示脚本过期
④填一个很大的数值时，可实现 "延长有效期"
⑤删除本字段，则认为脚本永久不会过期
⑥修改系统时间也可以绕过精灵的时间戳检查
⑦以上3个方法，只适用 "动作不加密" 的脚本
⑧加密的脚本，除了信息头，其他动作都加密了
⑨加密过的动作，运行时都会检查是否过期

④删除本字段，脚本就可以再次上传市场了
